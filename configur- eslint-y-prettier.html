Error

si en react con vite sale este error:

'React' must be in scope when using JSXeslintreact/react-in-jsx-scope

este error se debe ya que por defecto eslint esta comfigurado para que por defecto se tengamos que hacer un import React from 'react' en cada archivo.

Con Vite esto no es necesario ya que vite utiliza el runtime automatic ya que usa la nueva trasformacion desde react 17.

Para solucionar este error se puede o desabilitar la regla (no es la manera recomendada) en el archivo ".eslintrc"

rules: {
  'react/react-in-jsx-scope': 'off'
}

tambien se puede poner un pequeño plugin (manera recomendada) en el archivo ".eslintrc"

extends : [
  'plugin:react/recommended',
  'plugin:react/jsx-runtime',
  'standard'
]

prettier

Primero comenzamos instalando el paquete de prettier.

npm i -E -D prettier

configuraremos el prettier con su archivo respectivo .prettierrc

{
  "arrowParens": "avoid",
  "useTabs": true ,
  "singleQuote": true ,
  "jsxSingleQuote": true
}

compatibilidad eslint y prettier

para que ambos paquete sean compatibles tenemos que instalar un paquete oficial

npm i -D eslint-config-prettier

haora solo tenemos que llamarlo en el archivo .eslintrc

extends: [
  'plugin:react/recommended',
  'plugin:react/jsx-runtime',
  'standard',
  'eslint-config-prettier'
]

con esto ya esta todo comfigurado y eslin tomara las reglas de prettier por delante de la de el

ultima configuración

tenemos que configurar ambos paquetes para que ignore unas carpetas y archivo

para ignorar mediante prettier usamo el archivo .prettierignore y colocamos lo siguientes

dist
package-lock.json

en el caso de eslint usamos el archivo .eslintignore y solo ignoramos lo siguiente

dist

configuramos los script

para prettier ponemos

"format":"prettier --write ."

para eslint ponemos

"lint": "eslint --fix .js,.jsx"

en el script de eslint tenemos q colocar las extenciones de los achivos q qremos q afecten

Error comune

Warning: React version not specified in eslint-plugin-react settings. See https://github.com/jsx-eslint/eslint-plugin-react#configuration

solución: es poner en el archivo .eslintrc

settings: {
  react: {
    vercion: 'detect',
  },
}

archivos configurado completo

.eslintrc

module.exports = {
  env: {
      browser: true,
      es2021: true,
    },
  settings: {
    react: {
      version: 'detect',
    },
  },
  extends : [
    'plugin:react/recommended',
    'plugin:react/jsx-runtime',
    'standard',
    'eslint-config-prettier',
  ],
  overrides : [],
  parserOptions : {
    ecmaVersion : 'latest',
    sourceType : 'module',
  },
  plugins : ['react'],
  rules : {
    // esta regla es obsional para no tener que asignarles
    // los tipos a cada props de los componentes
    // 'react/prop-types': 'off' ,
  },
};

.prettierrc

  "useTabs" : true,
  "singleQuote" : true,
  "jsxSingleQuote" : true
